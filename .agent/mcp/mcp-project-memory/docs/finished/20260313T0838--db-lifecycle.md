# DB Lifecycle Management

> Created: 2026-03-13 08:38 (local)
> Status: Draft

## Requirement

### Lazy DB — No Connection During Extraction

- **What**: DB is opened eagerly by `create_components()` and held open for entire build (30+ minutes), even though it's only used at the end for `_rebuild_db()` and `BuildMetaStore.record()`
- **Where**: `project-memory` (entrypoint), `db.py`, `build.py`
- **Why**: Wastes file handles, leaves WAL/SHM files on disk during build, contradicts the "DB is disposable cache" philosophy
- **How**: Don't open DB until `_rebuild_db()`. Have `BuildAgent` create its own DB connection scope at the end.
- **Priority**: High
- **Effort**: Medium

### Close DB After Rebuild

- **What**: After `_rebuild_db()` writes the cache, close the connection immediately so WAL/SHM files are cleaned up
- **Where**: `build.py` `_rebuild_db()`, `project-memory` entrypoint
- **Why**: No reason to hold the connection after the cache is built — next user is the MCP server, which opens its own connection
- **How**: Use `Database` as a context manager (`with Database(...) as db:`) for scoped access
- **Priority**: High
- **Effort**: Low
